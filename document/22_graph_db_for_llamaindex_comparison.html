Kùzu

임베디드 (In-process)

  • 서버 구축 0%. pip install kuzu로 즉시 사용 (로컬 파일 기반)
  • LlamaIndex KuzuGraphStore 공식 지원 우수
  • 복잡한 분석 쿼리(Join 등)에 극도로 최적화됨
  • 분산 서버 아키텍처 지원 안함 (단일 노드 용)
LlamaIndex 활용도
PoC 및 데이터 사이언스 워크플로우 최적
Speed Demon
M

Memgraph

인메모리 (C++)

  • Neo4j의 완벽한 대체제 (Bolt 프로토콜, Cypher 완벽 호환)
  • 데이터를 메모리에 올려 밀리초(ms) 단위의 초고속 그래프 탐색
  • LlamaIndex의 Neo4j 플러그인을 그대로 재사용 가능
  • RAM 용량에 데이터 크기가 제한됨 (비용 문제)
LlamaIndex 활용도
저지연(Low-latency) 실시간 RAG 애플리케이션 최적
Enterprise Giant
N

NebulaGraph

분산 클러스터

  • 수십억 노드의 초대규모 데이터 분산 저장/처리 설계
  • LlamaIndex와 가장 긴밀하게 협업하는 파트너 (통합 모듈 최상)
  • 자체 쿼리(nGQL) 외에 OpenCypher 지원 추가
  • 초기 인프라 구축 및 유지보수 학습 곡선이 가파름
LlamaIndex 활용도
대기업 수준의 방대한 지식 그래프 프로덕션 최적
